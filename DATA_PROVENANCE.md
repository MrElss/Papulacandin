# Data provenance & reproducibility

This file centralizes where the curated data came from, how the computed
columns were generated, and which toolchain reproduces the results. It
complements the per-table notes in `curated/` and the per-phase write-ups in
`analysis/outputs/`.

## Curated main library (`curated/core_tables/`)

| Table | Rows | Contents |
|---|---|---|
| `compounds_master.csv` | 138 | one row per papulacandin-family compound |
| `activity_table.csv` | 1042 | curated whole-cell / activity records |
| `enzyme_assays.csv` | — | enzyme-level (FKS/glucan-synthase) assay records |
| `structure_quality_notes.tsv`, `structure_qc_manual_review.csv` | — | structure-source, stereochemistry and manual-QC notes |
| `synthesis_feasibility.csv` | — | synthesis / follow-up feasibility |
| `after_analysis_reference/sar_annotations.csv` | — | **after-analysis reference only** — literature/human SAR, deliberately quarantined so analysis does not start from it |

Structures are provided as `structures/{sdf,mol,cdx}/` (137 each; ChemDraw `cdx`
is the source, `mol`/`sdf` are exports for cheminformatics).

### Literature sources

Compounds trace to **25 distinct primary references** recorded per row in the
`reference_id` / `source_table_or_figure` columns (e.g. `PAPER-1977-TRAXLER`,
`PAPER-1979-BAGULEY`, `PAPER-1983-ROMMELE`, `PAPER-1987-TRAXLER`,
`PAPER-1991-VANMIDDLESWORTH`, `PAPER-1992-BARTIZAL`, the Yeung-1996 fusacandin
series, and later isolations through 2026). Each activity row additionally
carries `reference_id`, `source_table_or_figure`, and a `source_quote`.

### Computed structure columns

The structure/descriptor columns in `compounds_master.csv`
(`smiles_canonical`, `inchikey`, `mw_exact`, `clogp`, `tpsa`, `hbd`, `hba`,
`rotb`, `fsp3`, …) were **generated by RDKit from the curated SDF on
2026-06-10** (recorded in the per-row `notes`, present on 137/138 rows). Raw
literature values (formula, MW) are preserved alongside in the notes for
cross-checking. The `tests/` InChIKey round-trip guards these against silent
drift when the toolchain changes.

## The dependent variable (serum gap)

- **Endpoint:** whole-cell MIC vs *Candida albicans*, matched **serum-free** vs
  **serum**; the derived measures are the raw **serum MIC** and the
  **serum SHIFT fold** = serum MIC / serum-free MIC (the potency-independent
  measure of true serum tolerance).
- **Matched set:** 24 compounds (Yeung-1996 fusacandin analogs 6a–6u +
  Fusacandin A/B + Papulacandin B). 13/24 retain measurable serum activity;
  **11/24 serum MICs are censored at `>100`**, and values are tied on
  {12.5, 25, 50, 100}. This censoring/tying is the statistical ceiling on every
  correlation in the project.
- **Censoring handling:** censored values (`>100`, `<0.03`) are handled
  explicitly (survival/Tobit-style), not silently dropped — see the docstring of
  `analysis/serum_gap_analysis.py`.

## External FKS corpus (used by Phase 11)

`external/data/external/fks_inhibitors/source_exports/` holds ChEMBL/PubChem-
curated activity for other β-1,3-glucan-synthase inhibitors, including a deep
echinocandin set (caspofungin, anidulafungin, micafungin, cilofungin, echinocandin
B, aculeacin A, pneumocandin) and the tail-free triterpenoids ibrexafungerp /
enfumafungin. Phase 11 mines the **serum-context** rows — `Ratio` endpoints whose
`source_quote` reads "MIC in 50% (human/mouse) serum **to** MIC in serum-free
medium/RPMI 1640", i.e. a serum-shift fold in the same direction as
`serum_shift_fold` — plus `PPB` (plasma protein binding) and `Fu` (fraction
unbound) rows. Serum-shift folds are taken on standard reference isolates (not the
FKS-hotspot-mutant strains, which appear only under plain `MIC` endpoints) and
aggregated per compound by median. Descriptors for both chemotypes are recomputed
from SMILES with the same RDKit build so the read-across is method-consistent.

### External QM / docking binaries (not pip-installable)

Regenerating raw QM/docking data — not required for phases 0–9, which only parse
the outputs — needs, from a conda-forge / module / licensed install:

- **CREST** — conformer ensemble search (GFN-FF screening tier; GFN2 for finalists).
  Settings actually used are documented in `analysis/README.md`.
- **xtb** — GFN2-xTB single points (phase 9 electronics/solvation).
- **Gaussian** — B3LYP/6-31G(d), PCM water, Pop=MK DFT single points.
- **AutoDock Vina** (+ `meeko`, `vina` Python wrappers) — phase 10 HSA docking.

Bulky regenerable QM/docking artifacts are gitignored (see `.gitignore`); the
decision-relevant results (GFN2 ensembles, sbatch scripts, xtb `.charges`/`.out`,
scores CSVs) are tracked.

## Reproducing the fast path

```bash
python3 -m pip install -r requirements.txt
python3 analysis/serum_gap_analysis.py
python3 analysis/phase1_serum_shift_sar.py
python3 -m pytest tests/ -q
```

Outputs land in `analysis/outputs/`. The full ordered run (including QM phases,
which need the external binaries above) is documented in `analysis/README.md`.
